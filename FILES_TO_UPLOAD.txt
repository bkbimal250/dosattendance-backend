===============

scp test_push_functionality.py root@82.25.109.137:/var/www/EmployeeAttendance/
scp manual_test_commands.sh root@82.25.109.137:/var/www/EmployeeAttendance/

Method 2: Using the batch script:
Run: upload_testing_files.bat

Method 3: Using the shell script:
Run: ./deploy_testing_files.sh

AFTER UPLOADING:
================

1. SSH into your server:
   ssh root@82.25.109.137

2. Navigate to project directory:
   cd /var/www/EmployeeAttendance

3. Set permissions:
   chmod +x test_push_functionality.py
   chmod +x manual_test_commands.sh
   chmod +x verify_database_records.py
   chmod +x server_test_script.sh

4. Run the server test script:
   ./server_test_script.sh

5. Run automated tests:
   python3 test_push_functionality.py --production

6. Run database verification:
   python3 verify_database_records.py

7. Run manual tests:
   ./manual_test_commands.sh

MONITORING:
===========

Monitor logs in real-time:
- Apache2 Access: tail -f /var/log/apache2/device_push_access.log
- Apache2 Error: tail -f /var/log/apache2/device_push_error.log
- Django Logs: tail -f /var/www/EmployeeAttendance/logs/django.log

SUCCESS CRITERIA:
=================

Your push attendance system is working if:
✅ Health check endpoint returns 200 OK
✅ Push endpoint accepts POST requests
✅ Attendance records are created in database
✅ New devices are created automatically
✅ Real biometric device can push data
✅ Logs show successful processing

TROUBLESHOOTING:
================

If tests fail:
1. Check if push_views.py is uploaded
2. Check if users have biometric_id set
3. Check if server is running on port 8081
4. Check Apache2 configuration
5. Check database connection
6. Review error logs

For detailed troubleshooting, see: TESTING_GUIDE.md
